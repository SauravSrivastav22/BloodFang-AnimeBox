 CLOUD SYNC — Google sign-in (BUILT; awaiting user setup)
--------------------------------------------------------
 Goal: sign in with Google (Gmail) so data (My List, Continue
 Watching, watched episodes, settings) saves to the cloud and
 comes back on any other device you sign into. Merge = union, so
 existing local data is never lost.

  [x] Code complete  -- DONE 2026-07-15
        Firebase Auth (Google) + Firestore, all client-side (no
        backend change). Lazy-loaded (only downloads on Sign in).
        New files: src/firebase-config.js, src/sync.js,
        src/AccountButton.jsx, FIREBASE_SETUP.md. Stores emit
        bloodfang:datachange; sync pulls+merges on login then
        debounced-pushes on change. Sign-in button hidden until
        configured, so app stays local-only meanwhile.
        Lint + build green (firebase = separate lazy chunk).

  [ ] TODO (user, ~5 min, one time) — see FIREBASE_SETUP.md
        1. Create a free Firebase project (console.firebase.google.com)
        2. Register a Web app; paste the 6 config values into
           src/firebase-config.js (apiKey, authDomain, projectId,
           storageBucket, messagingSenderId, appId)
        3. Authentication → enable Google sign-in
        4. Firestore → create DB (production mode) + paste the
           security rules from FIREBASE_SETUP.md
        Then: npm run prod → Sign in (top-right) → data syncs.
        (Optional: Claude can fill in firebase-config.js if the
         user pastes the config values.)

 NOTE: phones sync DATA but can't PLAY (embeds need a local Node
 backend + localhost) — treat a phone as view/manage-list only.
